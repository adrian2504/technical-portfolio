# Amazon Q Developer Portfolio Workshop Prompt

## Workshop Concept
Instead of manually coding, participants use Amazon Q Developer to build their technical portfolio through guided conversation. This demonstrates AI-powered development while creating something personally valuable.

## Pre-Workshop Preparation

### Participant Checklist
Before the workshop, participants should gather:

- [ ] **Professional Bio** (2-3 paragraphs about their background and interests)
- [ ] **Resume/CV** (for career timeline and skills extraction)
- [ ] **Color Preferences** (favorite colors for personalization)
- [ ] **GitHub Username** (for repository links)
- [ ] **3-5 Key Projects** (with descriptions, technologies used, and outcomes)
- [ ] **Professional Photo** (optional, can use placeholder)
- [ ] **Contact Information** (LinkedIn, email preferences)

## The Master Prompt

### Initial Setup Prompt
```
I want to create a professional technical portfolio website using GitHub Pages. I'll provide you with my information, and I'd like you to help me build a complete 5-page portfolio with modern design and responsive layout.

Here's my information:

**Personal Information:**
- Name: [Your Full Name]
- Current Role: [Your Job Title]
- Professional Summary: [2-3 sentence elevator pitch]

**Background:**
[Paste your bio/resume content here]

**Projects to Showcase:**
1. Project Name: [Name]
   Description: [What it does]
   Technologies: [Tech stack]
   Impact/Outcome: [Results or learnings]
   GitHub Link: [URL if available]

2. [Repeat for 3-5 projects]

**Preferences:**
- Favorite Colors: [e.g., "blues and greens" or "pinks and purples"]
- GitHub Username: [username]
- LinkedIn: [profile URL]
- Contact Preference: [how you want to handle contact forms]

**Goals:**
- Target Audience: [who should visit this portfolio]
- Key Message: [what you want visitors to know about you]

Please create a complete portfolio with:
1. Homepage with hero section and featured projects
2. About page with career timeline
3. Projects page with detailed project showcases
4. Blog page (can have sample posts related to my interests)
5. Contact page with professional contact methods

Use modern, clean design with responsive CSS. Make it professional but reflect my personality through the color scheme.
```

## Workshop Flow

### Phase 1: Information Gathering (15 minutes)
- Participants fill out their information template
- Review examples of good portfolio content
- Discuss color scheme and branding choices

### Phase 2: Q Developer Interaction (45 minutes)
- Participants paste their customized prompt into Amazon Q Developer
- Q generates the complete portfolio structure
- Participants iterate on design and content with follow-up prompts

### Phase 3: Customization & Enhancement (30 minutes)
**Follow-up Prompts for Refinement:**

```
"The color scheme isn't quite right. I prefer [specific colors]. Can you update the CSS to use a [color1] to [color2] gradient for headers and [color3] for buttons and links?"
```

```
"Can you add a section to my projects page highlighting my open source contributions and community involvement?"
```

```
"I'd like to embed a YouTube video of my conference talk. Can you show me how to add that to my about page?"
```

```
"The contact form needs a backend. Can you suggest options for GitHub Pages and show me how to implement one?"
```

### Phase 4: Deployment (20 minutes)
```
"Now help me deploy this to GitHub Pages. Walk me through creating the repository and enabling Pages."
```
